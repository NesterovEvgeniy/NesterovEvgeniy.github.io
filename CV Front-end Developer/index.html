html5

  • свободно владею языком гипертекстовой разметки, в работе уверенно использую HTML5;
  • выполняю семантичесую верстку с использованием тегов: header, footer, section, main, nav, form, article и другие;
  • добавляю мультимедийный контент на сайт тегами: img, picture, figure, video, audio, iframe; а также работаю с векторной графикой SVG;
  • для повышения кроссбраузерности использую файлы в разных форматах : видео - .webm, .mp4, .ogv; аудио - .ogg, .mp3, .wav; шрифты - .ttf, .woff, .woff2, .eot;
  • повышаю доступность сайтов за счет: заполнения атрибутов alt (кроме декоративных элементов), использования файлов с разрешением .ftt и тега track для отображения субтитров и другое
  • до начала работы продумываю структуру сборки сайта (Information architecture), HTML-страниц с учетом необходимости, адаптивной верстки (responsive);
  • при построении структуры web-старницы использую методологию БЭМ (блок, элемент, модификатор):
  • - задаю максимально понятные имена классам, использую вложенность блоков и элементов,
  • - при построении файловой структуры файлы группирую по типу, что не соответствует БЭМ, но обеспечивает доступность и скорость работы (подход к работе меняю в зависимости от требований компании)
  • отладку HTML-кода выполняю сразу в процессе верстки (использую в Gulp: npm html-validator);
  • подключаю сторонние библиотеки, фреймворки, шрифты, favicon.

Bootstrap

  • в основном использую данный фреймворк для переноса необходимых элементов в свои проекты;
  • наиболее часто используемые элементы: карусель, выпадающие списки, навигация и другие;
  • использую для создания простых (шаблонных) лэндингов без кастомизации

css

  • уверенно использую каскадную таблицу стилей (CSS) для стилизации html-страниц;
  • в работе применяю flex, grid для построения элементов внутри контейнера;
  • создание анимаций при помощи свойств CSS (animation, transition) и различных библиотек;
  • адаптивная, отзывчивая верстка (responsive layout): использую в работе относительные единицы (%, em, rem, vw, vh), flexbox и медиа-запросы (@media)
  • выполняю кроссбраузерную верстку: применяю reset.css, normalize.css; использую автоматическую расстановку префиксов в таскраннере Gulp (плагин npm gulp-autoprefixer); выполняю проверку в различных браузерах, при необходимости ручная расстановка префиксов;
  • добавляю и настраиваю анимацию при помощи свойств animation и transition;
  • повышаю доступность сайтов за счет: использования visibility:hidden вместо display:none, где это необходимо; атрибут aria не приходилось использовать, при необходимости могу углубиться в данную тему;
  • отладку СSS-кода выполняю сразу в процессе стилизации (использую в Gulp: npm css-validator, npm gulp-plumber);
  • подключаю шрифты, использую переменные, применяю препроцессоры;

Gulp

  • для ускорения работы использую таскраннер Gulp;
  • имею собственную сборку, которую адаптирую под техническое задание каждого проекта;
  • обычно в работе использую следующие плагины:
  • npm gulp-sass - плагин для работы с препроцессором Sass;
  • npm gulp-concat - плагин для конкатинации и переименования файлов;
  • npm gulp-clean - плагин для удаления папок и файлов;
  • npm gulp-cssnano, npm gulp-uglify - плагины для минификации файлов css и js;
  • npm gulp-browser-sync - плагин для обновления страницы браузера;
  • npm gulp-autoprefixer - плагин для автоматической расстановки префиксов;
  • npm gulp-plumber - плагин для выявления и вывода ошибок, без остановки работы программы;
  • npm gulp-imagemin - плагин для сжатия и оптимизации изображений;
  • и многие другие.
